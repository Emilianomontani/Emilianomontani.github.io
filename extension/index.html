Extensión para verificar los errores básicos del criterio 1.3.1 Información y relaciones.

Esta aplicación está en fase beta. Puede contener fallos puntuales en los resultados lanzados. Se debe realizar una comprobación manual para verificar el contenido.

Se aplica un rol de estado para que la lista se anuncie a las personas usuarias que navegan con productos de apoyo. La extensión es funcional navegando con teclado e incluye un foco visible y un contraste de más de 4,5:1 en todos sus textos y elementos interactivos.

La opción para visualizar los errores en la página, colorea todos los errores de una vez, excepto los que están ocultos visualmente. Para ello aplica la clase "resaltado-error" y está se puede buscar en el código mediante el inspector para localizar los errores o etiquetas que no son visibles. Por otro lado este resaltado, dependiendo el fondo del elemento puede no cumplir constraste.

  • Verifica si hay párrafos vacíos o con " - - ".
  • Si hay etiquetas con <b>, <u>, <small>, <br>, <Font>.
  • Si hay etiquetas <div> o <span> con texto incluido directamente. Se indica que ignore si están dentro de enlaces, botones, <li>, encabezados y etiquetas con el role="button" o role="link", pero puede contener fallos por la complejidad de ciertas estructuras HTML. En la lista se genera un pequeño fragmento del código para que sea más fácil localizar el error o comprobarlo.
  • Si hay etiquetas <i> y si ellas tienen el atributo aria-hidden="true".
  • Si hay encabezados vacíos.
  • Si hay encabezados vacíos dentro de tablas.
  • Si hay más de un encabezado <h1>.
  • si hay listas vacías <ul> o <ol>.

Los errores se imprimen mediante un encabezado, un texto con el valor númerico de los errores encontrados y luego una lista detallada de errores. En muchos casos, esta lista puede ser extensa y repetitiva, pero fue la manera que encontré para que los errores se resaltaran correctamente en el código.

Para conocer más sobre este criterio o sobre las WCAG en general: Criterio 1.3.1 (WCAG2.2)
