Hola!

Esta extensión está diseñada para verificar automáticamente errores básicos relacionados con el criterio 1.3.1 (Información y Relaciones) de las WCAG.

Al presionar sobre el enlace, la extensión se descarga automáticamente. El peso de la extensión es de 140kb y está comprimida en formato zip

Importante: Fase beta

Por favor, ten en cuenta que esta aplicación se encuentra en fase beta. Los resultados generados podrían contener imprecisiones. Se recomienda realizar una verificación manual para confirmar la validez de los errores detectados.

Características de Accesibilidad de la Extensión:

  • Se aplica un role="status" al contenedor de la lista de errores para asegurar que se anuncie correctamente a usuarios de lectores de pantalla y otras tecnologías de asistencia.
  • La extensión es totalmente funcional mediante navegación con teclado, incorporando un indicador de foco visible y un contraste de color superior a 4.5:1 en todos sus textos y elementos interactivos.

Visualización de Errores en la Página:

La opción para visualizar los errores colorea todos los elementos problemáticos de una vez (excepto aquellos que están visualmente ocultos). Para lograr esto, se aplica la clase CSS "resaltado-error". Puedes utilizar el inspector de elementos de tu navegador para buscar esta clase en el código fuente y localizar los errores o etiquetas no visibles. Advertencia: El contraste del resaltado aplicado podría no cumplir con las WCAG dependiendo del color de fondo del elemento original.

verificaciones realizadas

La extensión actualmente verifica los siguientes aspectos:

  • Verifica si hay párrafos vacíos o con " - - ".
  • Si hay etiquetas con <b>, <u>, <small>, <br>, <Font>.
  • Si hay etiquetas <div> o <span> con texto incluido directamente. Se indica que ignore si están dentro de enlaces, botones, <li>, encabezados y etiquetas con el role="button" role="link" etc. Nota: Debido a la complejidad de ciertas estructuras HTML, podrían existir falsos positivos. Se incluye un fragmento del código del elemento detectado en la lista de errores para facilitar su localización y comprobación.
  • Si hay etiquetas <i> y si ellas tienen el atributo aria-hidden="true".
  • Si hay encabezados vacíos.
  • Si hay encabezados vacíos dentro de tablas.
  • Si hay más de un encabezado <h1>.
  • si hay listas vacías <ul> o <ol>.

Presentación de los Resultados:

Los errores detectados se presentan con un encabezado descriptivo de la verificación realizada, seguido de un texto que indica la cantidad total de errores encontrados para esa categoría y una lista detallada de cada error individual. Nota: En algunos casos, la lista de errores puede ser extensa y repetitiva debido a la forma en que se implementa el resaltado visual en el código.
